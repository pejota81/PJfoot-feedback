# MISTER v0.26.4 - Deep Admin Accounts Reliability Fix

> Released: 2026-05-17

This patch is a deep reliability fix for Admin Center account actions after repeated production 404 failures.

---

## Fixes

- **Account mutations in production:** fixed account type update and account deletion failing with `HTTP 404`.
- **Legacy/save-only account handling:** account mutation logic now works even when a user exists only in saves data and does not yet have a user document.
- **Account list persistence:** fallback account list data now carries role/status/account metadata from `/api/stats` when dedicated accounts APIs are unavailable, preventing account type from always appearing as `basic`.

## Technical

- Added production-safe mutation endpoints in `api/src/functions/admin/stats.js`:
  - `PATCH /api/admin-account-mutate`
  - `DELETE /api/admin-account-remove?userId=...`
- Enhanced stats payload user objects to include account metadata (`email`, `displayName`, `providers`, `roles`, `accountStatus`, `createdAt`, `lastLoginAt`).
- Updated mutation handler behavior in `stats.js`:
  - Accepts `userId` from route/body/query
  - Deletes save-only users even when no user document exists
  - Creates a minimal user document on update when needed, then applies role/status changes
- Updated `src/api/adminClient.ts` legacy/fallback mapping so `/api/stats` can preserve account role/status metadata in the UI.
